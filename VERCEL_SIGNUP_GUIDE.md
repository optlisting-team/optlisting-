# Vercel 가입 가이드

## 현재 상황

프론트엔드(React)를 Vercel에 배포해야 합니다.

---

## 플랜 선택

### Hobby (개인 프로젝트) - 무료

**선택 가능:**
- ✅ MVP/테스트 단계
- ✅ 개인 학습/포트폴리오
- ✅ 개발 중인 프로젝트
- ✅ 트래픽이 적은 초기 단계

**제한사항:**
- 개인 프로젝트로 명시되어야 함
- 상업적 목적이라면 나중에 Pro로 업그레이드 필요

### Pro (상업용) - 유료

**필요한 경우:**
- 상업적 서비스 운영
- 고객에게 유료 서비스 제공
- 높은 트래픽 예상
- 팀 협업 기능 필요

---

## 🎯 현재 프로젝트에 맞는 선택

### 추천: Hobby로 시작 ✅

**이유:**
1. **현재는 개발/테스트 단계**
   - 실제 상업적 운영 전 단계
   - MVP 배포 및 테스트 목적

2. **비용 절감**
   - 무료로 시작 가능
   - 필요시 나중에 Pro로 업그레이드

3. **기능 충분**
   - 무료 플랜으로도 프론트엔드 배포 충분
   - 커스텀 도메인, SSL 등 기본 기능 제공

---

## ⚠️ 주의사항

### Hobby 플랜 사용 시:
- ✅ "I'm working on personal projects" 선택
- ⚠️ 나중에 상업적 목적이 명확해지면 Pro로 업그레이드 필요
- ⚠️ Vercel 서비스 약관 확인

### 나중에 Pro로 업그레이드:
- 실제 사용자 유치 시
- 유료 서비스 시작 시
- 트래픽 증가 시

---

## 가입 단계

### 1. 플랜 선택
- **"I'm working on personal projects"** 선택 ✅
- Hobby 플랜으로 시작

### 2. Continue 클릭
- 계정 생성 진행

### 3. GitHub로 가입
- "Continue with GitHub" 권장
- `optlisting-team` 조직 저장소 접근 권한 허용

---

## 다음 단계

가입 후:
1. **프로젝트 생성**
   - "New Project" 클릭
   - `optlisting-team/optlisting-` 저장소 선택

2. **설정**
   - Root Directory: `frontend`
   - Build Command: `npm run build`
   - Output Directory: `dist`

3. **환경 변수**
   - `VITE_API_URL`: Railway 백엔드 URL
