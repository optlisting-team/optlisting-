# 기존 Railway 프로젝트 삭제 가이드

## 삭제 전 확인 사항

### ✅ 삭제해도 되는 경우:
- 새로 연결한 Railway 프로젝트가 정상 작동
- 기존 프로젝트에 중요한 데이터 없음
- 기존 프로젝트의 도메인/설정이 더 이상 필요 없음

### ⚠️ 삭제 전 확인:
- [ ] 새 Railway 프로젝트가 정상 배포되었는지 확인
- [ ] 새 프로젝트에 환경 변수 설정 완료
- [ ] 새 프로젝트에서 API 테스트 성공
- [ ] 기존 프로젝트에 중요한 데이터/설정이 없는지 확인

## Railway 프로젝트 삭제 방법

### 방법 1: Railway 웹 대시보드에서 삭제

1. **Railway 대시보드 접속**
   - https://railway.app 접속
   - 기존 프로젝트 선택

2. **Settings 탭 클릭**
   - 프로젝트 상단의 "Settings" 탭 클릭

3. **Danger Zone 찾기**
   - Settings 페이지 하단으로 스크롤
   - "Danger Zone" 섹션 찾기

4. **Delete Project 클릭**
   - "Delete Project" 버튼 클릭
   - 확인 대화상자에서 프로젝트 이름 입력
   - "Delete" 버튼 클릭

5. **삭제 확인**
   - 프로젝트가 목록에서 사라짐
   - ⚠️ **삭제 후 복구 불가능!**

### 방법 2: 서비스만 삭제 (프로젝트 유지)

프로젝트는 유지하고 서비스(web)만 삭제:

1. **서비스(web) 선택**
2. **Settings 탭 클릭**
3. **Delete Service 클릭**
4. **확인**

## 삭제 후 확인

### 새 프로젝트 확인:
- [ ] 새 Railway 프로젝트가 정상 작동하는지 확인
- [ ] API 문서 접속: https://web-production-3dc73.up.railway.app/docs
- [ ] `/api/listings` 엔드포인트 테스트
- [ ] 500 에러 없이 정상 응답 확인

## 주의사항

- ⚠️ **삭제 후 복구 불가능**
- ⚠️ **프로젝트 내 모든 서비스, 데이터, 설정이 삭제됨**
- ⚠️ **환경 변수도 모두 삭제됨**

## 권장 순서

1. **먼저 새 프로젝트 확인**
   - 새 Railway 프로젝트가 정상 작동하는지 확인
   - 환경 변수 설정 완료 확인
   - API 테스트 성공 확인

2. **그 다음 기존 프로젝트 삭제**
   - 새 프로젝트가 안정적으로 작동하는 것을 확인한 후
   - 기존 프로젝트 삭제

## 대안: 기존 프로젝트 비활성화

삭제가 걱정되면:
- 기존 프로젝트는 그대로 두고
- 새 프로젝트만 사용
- 나중에 필요하면 기존 프로젝트 삭제
